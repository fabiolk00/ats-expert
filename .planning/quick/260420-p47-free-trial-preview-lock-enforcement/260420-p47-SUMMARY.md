## Summary

Implemented Phase 47 free-trial preview lock enforcement so the synchronous generation path persists the same lock state it returns, and timeline/version/compare surfaces no longer expose real optimized or target resume snapshots when preview access is locked.

## Delivered

- Introduced a richer central `PreviewAccess` contract in `src/types/agent.ts`.
- Centralized preview access application and lock summaries in `src/lib/generated-preview/locked-preview.ts`.
- Added shared version/compare sanitization in `src/lib/cv/preview-sanitization.ts`.
- Fixed synchronous `generateBillableResume` so `generatedOutput.previewAccess` is applied to both the returned output and the persisted patch.
- Exposed `generatedOutput` through the dispatch contract and updated `smart-generation` to read the guaranteed enriched result.
- Protected `session/[id]/versions` and `session/[id]/compare` from leaking real snapshots for locked free-trial previews.
- Added route and helper regression tests for locked persistence, versions, compare, and workspace session reads.
