## Summary

Added the consolidation layer for preview-lock enforcement: one transverse regression test for the full business flow, one signed URL audit snapshot, and one canonical preview lock policy document.

## Delivered

- Added `src/app/api/preview-lock-transverse.test.ts` to prove:
  - trial generation persists a locked preview
  - replay stays locked
  - versions hide real snapshots
  - compare hides real diffs
  - file route does not emit a real signed URL
  - upgrade alone does not unlock the old artifact
  - a new paid generation unlocks real content
- Hardened `assertNoRealArtifactForLockedPreview(...)` so locked responses also require persisted patch preview access.
- Documented signed URL emitters and their safety status in `260420-q71-SIGNED-URL-AUDIT.md`.
- Added `docs/preview-lock-policy.md` as the canonical product and technical policy.
