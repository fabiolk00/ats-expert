# Quick Task 260502-op1: Fix Shadow Batch Smoke Runner Server-Only CLI Imports

**Date:** 2026-05-02
**Status:** Completed

## Goal

Allow the Job Targeting shadow batch smoke runner to execute from a plain Node/tsx CLI without tripping Next.js `server-only` import guards, while preserving server-only boundaries in the application runtime.

## Context

The synthetic 100-case smoke run initially failed before processing cases because the CLI loaded modules that eventually imported `server-only`. This happened even for a deterministic non-persistent smoke run.

## Tasks

1. Avoid importing persistence helpers at shadow runner module load time when `--persist` is not enabled.
2. Avoid importing Supabase admin at usage-tracker module load time; load it only when persisting API usage entries.
3. Install a CLI-only `server-only` shim before dynamically importing the batch runner and analyzer helpers.
4. Re-run the synthetic 100-case smoke batch and analyzer.
5. Verify with typecheck and automated tests.
