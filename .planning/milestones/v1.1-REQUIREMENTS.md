# Requirements: CurrIA

**Defined:** 2026-04-10
**Core Value:** A job seeker can reliably turn their real profile and a target role into an honest, ATS-ready resume output they can confidently download and use.

## v1.1 Requirements

### Runtime Provenance

- [x] **OPS-04**: Operator can identify which build or commit served a real `/api/agent` request.
- [x] **OPS-05**: Completed `/api/agent` turns log the selected model, assistant text length, recovery usage, and fallback branch.
- [x] **OPS-06**: Team can run a documented post-deploy check that confirms live `/api/agent` config and runtime parity after rollout.

### Dialog Continuity

- [x] **AGNT-01**: User can send a follow-up rewrite request in `dialog` and receive a concrete rewrite or a non-repetitive continuation response.
- [x] **AGNT-02**: Dialog and confirm turns use the resolved model contract, including explicit `OPENAI_DIALOG_MODEL` overrides when present and agent-model fallback when absent.
- [x] **AGNT-03**: Truncation and empty-response recovery preserve the latest rewrite intent and target-job context instead of reverting to stale bootstrap behavior.

### Transcript Integrity and Verification

- [x] **UX-01**: One chat request renders as one coherent assistant turn in the visible transcript, even when recovery or fallback paths are used.
- [x] **QA-04**: Automated verification covers live-route model selection, dialog recovery, and the final rendered assistant output for rewrite flows.
- [x] **QA-05**: Team can reproduce and inspect a representative `reescreva` incident with committed route or transcript evidence.

## v1.2 Requirements

### Guided Rewriting Experience

- **AGNT-04**: User can request multi-section rewriting with explicit section-choice guidance in the chat flow.
- **UX-02**: Chat transcript can expose user-friendly recovery annotations when a response was degraded but still usable.

### Deployment Guardrails

- **OPS-07**: CI or deploy checks block promotion when `/api/agent` provenance or required log fields drift from the documented contract.

### ATS Enhancement Pipeline

- [x] **ATS-01**: A resume-only session deterministically resolves to `ats_enhancement` and persists ATS analysis plus optimized resume state before optional chat branching.
- [x] **ATS-02**: ATS enhancement rewrites the full supported resume structure into a validated `optimizedCvState` without inventing employers, dates, certifications, tools, or unsupported claims.
- [x] **ATS-03**: ATS-enhancement versions are persisted as first-class artifacts and preview or export flows prefer the latest valid optimized snapshot when available.

### ATS Enhancement Reliability

- [x] **REL-01**: ATS enhancement persists stage-aware execution state and bounded retries so large or partially failing rewrites do not degrade into opaque all-or-nothing failures.
- [x] **REL-02**: ATS rewrite validation catches both invented facts and cross-section inconsistencies so summary, experience, and skills stay mutually consistent.
- [x] **REL-03**: ATS-enhancement runs emit structured workflow observability and preserve the last valid optimized export source even after a newer rewrite attempt fails.

## v1.3 Requirements

### AI Resilience and Processing

- [x] **OPS-08**: OpenAI-backed routes fail fast with bounded retries, timeout budgets, and a circuit breaker so provider degradation cannot cascade across resume generation flows.
- [x] **OPS-09**: PDF extraction and parsing do not run as an unbounded synchronous request path when payload size or provider latency would risk route-level timeout.

### Security and Compliance

- [x] **SEC-01**: The platform documents and enforces LGPD-sensitive data handling for resume, vacancy, and generated artifact flows, including retention and deletion rules.
- [x] **SEC-02**: Test-only auth bypass and server secrets stay isolated behind explicit environment guards, with committed verification that they cannot leak into non-test surfaces.

### Engineering Quality

- [x] **ENG-01**: The repo exposes an enforceable TypeScript-first lint and formatting baseline instead of minimal default linting.
- [x] **QA-06**: Automated testing layers, command entrypoints, and expected coverage responsibilities are documented and visible to contributors.

### Maintainability

- [x] **ARCH-01**: Session persistence and quota orchestration are split into narrower modules so creation, patching, quota, and message operations can evolve and be tested independently.

## v1.4 Requirements

### Security Boundary Verification

- [x] **SEC-03**: Middleware, file access routes, webhook verification, and service-role boundaries are auditable end to end with committed proof of the effective protection model.
- [x] **SEC-04**: Test-only auth seams and file-access paths fail closed under unsupported runtime, ownership, or token conditions.

### Billing and Webhook Safety

- [x] **BILL-01**: Billing settlement, duplicate webhook delivery, and credit mutation invariants are explicitly verified so payment-side regressions cannot silently over-credit or expose paid artifacts.
- [x] **BILL-02**: Sensitive billing and secret-bearing provider paths are isolated behind documented server-only boundaries with route-level verification.

### Data Model Evolution

- [x] **DATA-01**: High-churn JSON persistence areas have explicit contracts, ownership, and reduction plans so stable data can migrate toward narrower schemas over time.
- [x] **DATA-02**: Resume, session, and targeting persistence can be queried and validated without relying on opaque JSON blobs for every core invariant.

## Out of Scope

| Feature | Reason |
|---------|--------|
| PDF and DOCX profile upload onboarding | Still deferred from the previous milestone; it does not address the current agent continuity incident |
| Billing or pricing changes | v1.0 already validated billing behavior; this milestone is about agent reliability and transcript trust |
| Broad UI redesign of the dashboard chat | Visual polish is secondary to proving and fixing runtime behavior |
| Full prompt-system rewrite | Too broad for the current bug; keep the milestone focused on proven failure seams |
