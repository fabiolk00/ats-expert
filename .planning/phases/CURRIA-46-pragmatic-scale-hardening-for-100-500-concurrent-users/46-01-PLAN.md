<objective>
Harden CurrIA's current export and billing path for moderate concurrency without redesigning the platform.
</objective>

<success_criteria>
- web nodes stop executing heavy export work inline
- worker nodes process queued export jobs with bounded local concurrency
- same-user export fanout is capped at one active export
- export timeout and bounded retry are explicit
- reserve/finalize/release/reconcile/render stages emit timing logs
- critical export and billing events emit machine-readable counter events
- hot-path indexes are added without schema redesign
- operators have one internal health page plus actionable alert summary logic
- staging validation commands are documented with explicit pass/fail thresholds
</success_criteria>
