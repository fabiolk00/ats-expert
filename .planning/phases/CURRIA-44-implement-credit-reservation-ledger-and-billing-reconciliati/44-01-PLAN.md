<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app billing service -> Postgres RPCs | Credit mutation crosses into financial state and must stay atomic |
| reservation records -> export runtime | Runtime retries must not create duplicate holds or contradictory transitions |
| ledger reads -> operator diagnostics | Audit views must remain append-only and attributable even if generation history drifts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-01 | T | reservation transitions | mitigate | Use unique generation-intent keys and RPC-backed state transitions so retry races cannot double-hold or double-release credits. |
| T-44-02 | R | ledger history | mitigate | Write append-only ledger entries for hold, finalize, and release with stable timestamps and intent metadata. |
| T-44-03 | E | reservation lookup/read paths | mitigate | Scope rows by `user_id` and keep mutation wrappers behind service-role DB access only. |
| T-44-04 | D | missing schema or RPC drift | mitigate | Fail closed in wrappers with explicit structured errors; do not silently skip billing mutations. |
</threat_model>
