## Objective

Add a dynamic, domain-agnostic semantic evidence layer for `job_targeting` targeted rewrite only, preserving existing targeting-plan compatibility fields while enabling:

- direct factual claims
- normalized aliases
- technical equivalents
- strong contextual bridges
- semantic related-context bridges
- explicit unsupported gaps

## Task 1: Create the semantic evidence contract and classifier

Arquivos:

- `src/types/agent.ts`
- `src/lib/agent/job-targeting/semantic-normalization.ts`
- `src/lib/agent/job-targeting/evidence-classifier.ts`
- `src/lib/agent/job-targeting/rewrite-permissions.ts`
- `src/lib/agent/tools/build-targeting-plan.ts`
- `src/lib/agent/tools/build-targeting-plan.test.ts`

Acoes:

- add `EvidenceLevel`, `RewritePermission`, `TargetEvidence`, and compact permission buckets to the targeting contract;
- classify job signals against original resume evidence with deterministic lexical matching first and semantic fallback second;
- keep the architecture domain-agnostic by centralizing only generic normalization rules in core code;
- preserve `focusKeywords`, `mustEmphasize`, `shouldDeemphasize`, and `missingButCannotInvent` for compatibility while adding `targetEvidence` and `rewritePermissions`.

Done:

- `buildTargetingPlan()` becomes the canonical owner of targeted semantic evidence without changing ATS callers.

## Task 2: Integrate permission-aware prompting and target-only validation

Arquivos:

- `src/lib/agent/job-targeting/validation-policy.ts`
- `src/lib/agent/tools/rewrite-resume-full.ts`
- `src/lib/agent/tools/validate-rewrite.ts`
- `src/lib/agent/job-targeting-pipeline.ts`
- `src/types/trace.ts`
- `src/lib/agent/tools/validate-rewrite.test.ts`
- `src/lib/agent/tools/pipeline.test.ts`

Acoes:

- feed `job_targeting` prompts with direct, normalized, bridge, contextual-only, and forbidden claim buckets;
- keep ATS and generic rewrite branches unchanged;
- relax validation only where target evidence explicitly permits normalization/equivalence;
- keep unsupported or over-inflated claims blocked;
- enrich `job_targeting` trace/log extraction with compact evidence counts only.

Done:

- targeted rewrite and targeted validation become evidence-aware without leaking to ATS or highlight-only flows.

## Task 3: Prove isolation and no-target compatibility

Arquivos:

- `src/lib/agent/job-targeting/evidence-classifier.test.ts`
- `src/lib/agent/tools/build-targeting-plan.test.ts`
- `src/lib/agent/tools/validate-rewrite.test.ts`
- `src/lib/agent/tools/pipeline.test.ts`

Acoes:

- cover explicit, normalized alias, technical equivalent, strong contextual inference, semantic bridge, and unsupported gap;
- prove bridge/context claims behave differently by surface;
- prove seniority inflation is still blocked;
- prove ATS, no-target, and shared session surfaces continue to pass their current focused suites.

Done:

- the semantic layer is executable proof, not only prompt prose.

## Isolation Notes

- The new layer runs only in `job_targeting` with active targeted rewrite.
- `rewrite-resume-full.ts` reads permission buckets only in the `job_targeting` branch.
- `validate-rewrite.ts` applies the new claimability policy only when `context?.mode === 'job_targeting'`.
- `highlightState` generation and non-target readers remain independent from the new evidence layer.
